# SEO.ai Phase 3B: Workflow and Prioritization

## Overview
Phase 3B makes deterministic recommendations operationally manageable.

Recommendations are still generated by the Phase 3A deterministic engine from persisted SEO evidence. Phase 3B adds workflow metadata, safe update actions, and deterministic prioritization views.

## Scope Implemented
- recommendation workflow state updates (`open`, `in_progress`, `accepted`, `dismissed`, `snoozed`, `resolved`)
- workflow decision tracking (`accept`, `dismiss`, `snooze`, `resolve`, `reopen`, `start`)
- assignee, due date, snooze, and resolution metadata
- site-scoped recommendation list filters/sorts
- deterministic backlog endpoint for actionable items
- deterministic prioritized report endpoint for rollups and backlog visibility

## Deterministic Boundary
Phase 3B does not generate new recommendation content and does not call AI.

All workflow and report behavior operates on persisted recommendation records and persisted deterministic fields:
- `priority_score`
- `priority_band`
- `severity`
- `category`
- `effort_bucket`
- workflow state metadata

## Workflow Rules
Implemented service-layer rules include:
- valid status transitions only
- decision/status consistency checks
- `snoozed` requires a future `snoozed_until`
- `snoozed_until` rejected for non-snoozed statuses
- assignee must resolve to an active principal in the same business
- cross-business and cross-site updates are rejected

## Prioritization Views
Backlog and prioritized report endpoints are deterministic and reproducible from stored records.

- backlog includes actionable items (`open`, `in_progress`, `accepted`) and resumed snoozed items whose `snoozed_until` has passed
- prioritized report includes aggregate counts and a stable backlog slice

## Out of Scope in Phase 3B
- AI recommendation narratives
- recommendation generation by LLM
- schedulers/automation/work queues
- broad task-management subsystem redesign

Future phases:
- Phase 3C: AI recommendation narratives over persisted recommendation evidence
- Phase 4: automation and operationalization
